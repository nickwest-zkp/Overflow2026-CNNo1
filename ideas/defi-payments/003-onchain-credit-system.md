# 链上信用系统 / On-Chain Credit Scoring

> 基于链上交易历史建立信用评分，信用高的用户可获得更低抵押率的贷款

## 解决什么问题

传统 DeFi 借贷对所有用户采用相同的抵押率要求，忽略了用户在链上的历史行为和信用积累。长期良好行为的用户被迫提供与新手相同的超额抵押，资金效率低下。链上信用系统通过分析用户的交易历史、还款记录和资产持有情况，生成链上可验证的信用评分，让高信用用户享受更低的抵押率。

## 核心功能

- 信用评分引擎：分析链上交易频率、还款准时率、资产多样性等维度，生成 300-900 分的信用评分
- 信用加权抵押率：信用分越高，借贷抵押率越低，最低可降至 105%
- 信用凭证 NFT：评分结果铸造为 Soulbound Token，不可转让，作为信用证明
- 信用报告生成：生成链上可验证的信用报告，可跨协议使用

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | CreditScore 作为独立 Object，绑定用户地址 |
| 动态字段 | 存储信用历史记录、评分因子和权重参数 |
| PTB | 原子化执行"查询信用 → 计算抵押率 → 创建贷款"复合操作 |
| Soulbound Token | 信用凭证 NFT 不可转让，保证信用绑定本人 |
| 共享对象 | 信用注册表作为共享对象，允许协议和用户读写 |

## 实现方案

### 智能合约 (Move)
- `credit_registry` 模块：管理用户信用注册和评分存储
- `score_engine` 模块：多维评分算法，包含链上行为分析器
- `credit_sbt` 模块：铸造和管理不可转让的信用凭证
- `credit_oracle` 模块：为借贷协议提供信用评分查询接口
- 关键数据结构：`CreditProfile { score: u16, factors: vector<ScoreFactor>, history: vector<CreditEvent>, updated_at: u64 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Credit Dashboard（信用总览）、Score Breakdown（评分因子详情）、Linked Loans（关联贷款）
- 后端索引链上历史交易，计算评分因子并提交到合约

## 技术架构

用户首次使用时，后端索引器扫描其链上历史（还款记录、交易频率、资产持有），计算评分因子并通过 PTB 提交到信用注册表。合约中的 score_engine 综合各因子生成最终评分并铸造 SBT。借贷协议通过 credit_oracle 模块查询用户评分，动态调整抵押率。评分每月自动更新，用户可通过 PTB 发起手动重算。所有评分变更通过事件通知前端和相关协议。
