# DAO 金库管理系统 / DAO Treasury Management

> 多签 + 规则引擎管理 DAO 资金，支持预算分配、支出审批、自动报表

## 解决什么问题

DAO 资金管理面临透明度不足、审批流程低效和缺乏自动化的问题。传统多签钱包只能做到"同意或拒绝"，无法实现复杂的预算规则和支出限制。DAO 金库管理系统将预算规则编码到 Move 合约中，结合多签机制和规则引擎，实现自动化的资金审批、预算控制和透明的财务报表。

## 核心功能

- 多层审批规则：根据金额大小自动匹配审批层级（小额自动、中额单签、大额多签）
- 预算分配引擎：按类别设置月度预算上限，超支自动拒绝
- 定期支付自动化：工资、订阅等定期支出自动执行，无需重复审批
- 链上财务报表：所有收支记录在链上，自动生成可验证的财务报告

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | Treasury 为共享对象，BudgetCategory 为嵌套对象 |
| PTB | 原子化执行"提案 → 审批 → 资金转移"复合操作 |
| 多签 (MultiSig) | 大额支出需要多个签名者确认 |
| 动态字段 | 存储预算分类、审批记录和支出历史 |
| 事件 (Events) | 发出收支事件，供索引器生成报表 |

## 实现方案

### 智能合约 (Move)
- `treasury` 模块：管理 DAO 资金池，处理存入和支出
- `budget_engine` 模块：定义预算分类和限额，检查支出合规性
- `proposal` 模块：创建和投票表决支出提案
- `recurring_payment` 模块：管理定期自动支付的注册和执行
- 关键数据结构：`Treasury { balances: Bag, budgets: vector<BudgetCategory>, signers: vector<address>, threshold: u8 }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + TailwindCSS + Recharts
- 关键页面：Treasury Overview（资金总览）、Proposals（提案列表）、Budget Settings（预算配置）、Reports（财务报表）
- 后端使用 Sui Event Indexer 聚合链上收支数据生成报表

## 技术架构

DAO 部署时创建 Treasury 共享对象，设置多签阈值和预算分类。成员通过 PTB 提交支出提案，合约自动检查是否在预算内：小额（< 阈值）直接执行，大额进入多签投票。预算引擎追踪每个分类的累计支出，超限自动拒绝。定期支付在注册后由后端 Keeper 按时触发 PTB 执行。所有收支记录通过事件发出，后端索引器聚合成可读的财务报表展示在前端。
