# 去中心化仲裁支付 / Decentralized Arbitration Payment

> 买卖双方存入托管，争议时仲裁员通过链上投票决定资金归属

## 解决什么问题

P2P 交易中争议解决依赖中心化平台（如支付宝、Upwork），仲裁过程不透明且费用高昂。用户对仲裁结果缺乏信任，平台可能偏袒一方。去中心化仲裁支付将托管资金和仲裁流程都放在链上，随机选取仲裁员投票决定资金归属，过程完全透明、结果不可篡改。

## 核心功能

- 双边托管合约：买卖双方各自锁定条件，资金在争议解决前不可动用
- 随机仲裁员选取：从质押池中随机选取 3-5 名仲裁员，防止串谋
- 链上投票仲裁：仲裁员提交证据哈希和投票，多数决定资金归属
- 仲裁员质押激励：仲裁员需质押代币，诚实投票获得奖励，恶意投票被 slashing

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | DisputeCase 作为独立 Object，存储争议详情和投票 |
| PTB | 原子化执行"存入托管 → 开启争议 → 投票 → 释放资金" |
| 共享对象 | 仲裁员注册池和争议案件作为共享对象 |
| Coin 标准 | 托管资金和仲裁员质押使用 Coin<T> |
| 动态字段 | 存储仲裁员投票、证据哈希和判决记录 |

## 实现方案

### 智能合约 (Move)
- `escrow` 模块：创建双边托管合约，锁定买卖双方资金
- `arbitrator_pool` 模块：管理仲裁员注册、质押和选取
- `dispute` 模块：创建争议案件，收集投票，执行判决
- `incentive` 模块：仲裁员奖励分配和 slashing 逻辑
- 关键数据结构：`DisputeCase { escrow_id: ID, arbitrators: vector<address>, votes: vector<Vote>, evidence_hashes: vector<vector<u8>>, deadline: u64 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Create Escrow（创建托管）、Dispute Center（争议中心）、Arbitrator Dashboard（仲裁员面板）、Vote Panel（投票界面）
- 后端监听争议事件，通知被选中的仲裁员参与投票

## 技术架构

交易开始时，买卖双方通过 PTB 将资金锁定到共享托管对象中。如果发生争议，任一方可发起争议案件，合约从仲裁员池中随机选取仲裁员并锁定其质押。仲裁员查看证据哈希后通过 PTB 提交投票，多数票决定资金归属。合约自动执行判决：胜诉方获得资金，仲裁员获得奖励。恶意投票的仲裁员被 slashing。所有过程通过事件通知前端。
