# 抵押品自动管理 / Auto Collateral Manager

> 当抵押率低于阈值时，PTB 自动补充抵押品或部分清算，避免硬清算

## 解决什么问题

DeFi 借贷中，价格剧烈波动会导致抵押率骤降，用户来不及补充抵押品就被硬清算，面临高额罚金和资产损失。现有协议的清算机制对借款人不友好，通常以折扣价拍卖抵押品。抵押品自动管理系统持续监控用户仓位健康状态，在危险时自动执行补充抵押或渐进式去杠杆，避免硬清算带来的损失。

## 核心功能

- 仓位健康监控：实时追踪所有借贷仓位的抵押率和健康因子
- 自动补充抵押：从用户授权的资金中自动转入抵押品，维持安全抵押率
- 渐进式去杠杆：当无法补充时，自动小额还款降低借贷量而非全额清算
- 紧急保护模式：极端行情下自动平仓部分仓位锁定最大损失

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | CollateralManager 作为独立 Object，持有用户授权和策略 |
| PTB | 原子化执行"检查健康 → 补充抵押/部分还款 → 更新仓位" |
| 动态字段 | 存储用户策略参数、阈值设置和执行历史 |
| 预言机集成 | 通过 Pyth 获取实时资产价格计算抵押率 |
| 事件 (Events) | 发出策略执行事件，通知用户保护操作已触发 |

## 实现方案

### 智能合约 (Move)
- `collateral_manager` 模块：注册和管理用户的自动保护策略
- `health_monitor` 模块：计算仓位健康因子和触发阈值判断
- `auto_action` 模块：执行补充抵押、渐进去杠杆、紧急平仓等操作
- `strategy_config` 模块：用户自定义策略参数（阈值、优先级、最大授权额度）
- 关键数据结构：`ProtectionStrategy { loan_id: ID, top_up_source: Option<Coin>, min_health_factor: u64, action_type: u8, max_auto_amount: Coin<T> }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Strategy Setup（策略配置）、Portfolio Health（仓位健康总览）、Action Log（自动执行日志）、Alerts（预警通知）
- 后端 Keeper 服务定期调用合约健康检查函数

## 技术架构

用户在借贷协议开仓后，通过 PTB 注册自动保护策略到 CollateralManager 对象中，授权合约在特定条件下动用指定资金。后端 Keeper 每个 epoch 检查用户仓位健康状态，当抵押率接近阈值时通过 PTB 执行保护操作：优先从授权资金中补充抵押，其次执行渐进式部分还款。所有操作通过事件记录，前端实时展示保护动作日志。用户可随时调整策略参数或取消自动保护。
