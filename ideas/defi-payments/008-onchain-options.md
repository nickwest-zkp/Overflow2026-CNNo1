# 链上期权协议 / On-Chain Options Protocol

> 基于 Sui 对象模型构建期权合约，行权、到期、清算全自动

## 解决什么问题

DeFi 期权市场目前规模远小于现货和借贷，主要原因是现有期权协议架构复杂、Gas 费高、用户体验差。传统期权合约需要中心化清算所管理行权和结算，效率低且不透明。基于 Sui 对象模型，每份期权合约作为独立对象存在，行权和到期逻辑完全编码在 Move 合约中，实现完全自动化和无需信任的期权交易。

## 核心功能

- 期权代币化：每份期权铸造为可转让的 NFT，买方拥有对象所有权
- 自动行权：到期时如果是价内期权，合约自动执行行权结算
- 期权金池：卖方存入抵押品到共享池中，赚取期权金收益
- 组合策略支持：支持构建价差、跨式、蝶式等组合策略

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | OptionContract 作为独立 Object，包含行权价、到期日、类型 |
| PTB | 原子化执行"行权 → 资产转移 → 期权销毁"复合操作 |
| Coin 标准 | 期权抵押品和结算使用 Coin<T> |
| 共享对象 | 期权抵押池作为共享对象，允许多卖方存入 |
| 时间锁 (Epoch) | 基于时间戳实现期权到期判断 |

## 实现方案

### 智能合约 (Move)
- `options_factory` 模块：创建新的期权系列（标的资产、行权价、到期日）
- `option_token` 模块：铸造和管理期权 NFT，处理行权逻辑
- `collateral_pool` 模块：管理卖方抵押品，计算抵押率
- `settlement` 模块：到期时自动结算，处理价内和价外情况
- 关键数据结构：`OptionContract { underlying: Type, strike: u64, expiry: u64, option_type: u8, amount: Coin<T>, writer: address }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Options Market（期权市场）、My Options（我的期权）、Write Options（卖出期权）、Portfolio P&L（盈亏分析）
- 后端监听期权事件，计算隐含波动率和 Greeks

## 技术架构

卖方通过 PTB 存入抵押品到共享池中并指定期权参数，合约铸造 OptionContract 对象。买方从市场购买期权后拥有对象所有权。到期时 Keeper 通过 PTB 触发结算：价内期权自动行权，抵押品按行权价转移给买方；价外期权自动过期，抵押品退还卖方。期权 NFT 可在二级市场自由转让。前端展示实时定价和到期日历，帮助用户管理期权组合。
