# 流式支付钱包 / Streaming Payment Wallet

> 按秒计费的实时支付，工资、订阅、租金都以流的方式传输，余额实时生息

## 解决什么问题

传统支付按月或按次结算，工资按月发、订阅按月扣、租金按月付，这种粗粒度的结算方式导致资金使用效率低下。员工要等一个月才能拿到工资，房东要等一个月才能收到租金。流式支付钱包让资金像水流一样按秒实时传输，发送方和接收方余额实时更新，未结算资金还可存入 DeFi 协议赚取收益。

## 核心功能

- 实时流式转账：创建支付流后，资金按设定速率（如每月 3000 USDC）每秒自动累积
- 余额生息：流中未结算的资金自动存入收益协议，利息按比例分配给双方
- 流暂停与恢复：发送方可随时暂停、恢复或调整流速
- 批量流支付：企业可一次创建数百条工资流，通过 PTB 批量执行

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | PaymentStream 作为独立 Object，存储流参数和累计金额 |
| PTB | 原子化执行"创建流 → 锁定资金 → 链接收益协议"复合操作 |
| Coin 标准 | 流资金使用 Coin<T>，支持任意代币 |
| 动态字段 | 存储流速、已提取金额、利息分配参数 |
| 共享对象 | 资金池作为共享对象，支持多方并发提取 |

## 实现方案

### 智能合约 (Move)
- `stream_factory` 模块：创建支付流，锁定发送方资金
- `stream_engine` 模块：计算实时余额、处理提取请求
- `yield_integration` 模块：对接借贷协议（如 Scallop），将闲置资金投入生息
- `batch_stream` 模块：批量创建和管理多条流
- 关键数据结构：`PaymentStream { sender: address, recipient: address, rate_per_second: u64, deposited: Coin<T>, withdrawn: Coin<T>, start_time: u64, stop_time: u64 }`

### 前端 / 后端
- 技术栈：React Native + @mysten/dapp-kit + Recharts
- 关键页面：Dashboard（资金流总览）、Create Stream（创建流）、Active Streams（活跃流列表）、Earnings（收益追踪）
- 前端实时计算并展示流的进度条和已到账金额

## 技术架构

发送方通过 PTB 创建支付流，指定接收方、流速和总金额，资金锁定在 PaymentStream 对象中。合约按秒计算已累积金额，接收方可随时通过 PTB 提取已累积部分。闲置资金通过 yield_integration 自动存入借贷协议赚取利息，利息按发送方和接收方的资金占比分配。发送方可暂停或取消流，未累积部分退还。前端使用定时器实时展示流的进度。

## 难度评估

| 维度 | 难度 | 说明 |
|------|------|------|
| Move 合约 | ⭐⭐⭐ | 流计算逻辑清晰，但生息集成增加复杂度 |
| 前端 | ⭐⭐⭐ | 实时进度展示需要流畅的动画和定时更新 |
| 集成复杂度 | ⭐⭐⭐ | 需要对接借贷协议实现闲置资金生息 |
