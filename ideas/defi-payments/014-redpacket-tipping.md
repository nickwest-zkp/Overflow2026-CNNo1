# 红包/打赏协议 / Red Packet & Tipping Protocol

> 创建链上红包，通过链接或二维码分享，领取后自动到账

## 解决什么问题

加密货币的社交转账体验远不如微信红包和打赏功能便捷。用户需要知道对方的钱包地址才能转账，无法像微信红包一样通过链接或二维码分享。红包/打赏协议将链上转账封装为社交化的红包体验，创建者设置金额和数量，生成分享链接或二维码，领取者点击即可将资金自动到账钱包。

## 核心功能

- 随机红包：总金额固定，每人领取随机金额（拼手气红包）
- 等额红包：每人领取固定金额
- 专属红包：指定接收地址才能领取
- 打赏按钮：网站/App 嵌入打赏组件，读者一键打赏创作者

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | RedPacket 作为独立 Object，存储资金和领取状态 |
| PTB | 原子化执行"验证资格 → 计算金额 → 转账 → 更新状态" |
| Coin 标准 | 红包金额使用 Coin<T>，支持任意代币 |
| 动态字段 | 存储领取记录（地址、金额、时间） |
| TransferPolicy | 控制红包的领取权限和转让规则 |

## 实现方案

### 智能合约 (Move)
- `redpacket_factory` 模块：创建红包，锁定资金并设置参数
- `claim_engine` 模块：处理领取逻辑，支持随机和等额两种模式
- `tipping` 模块：创建打赏链接和嵌入式组件
- `redpacket_nft` 模块：为特殊红包（如节日主题）铸造纪念 NFT
- 关键数据结构：`RedPacket { creator: address, total_amount: Coin<T>, remaining: Coin<T>, count: u8, claimed: u8, mode: u8, expire_time: u64 }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + QRCode 生成库
- 关键页面：Create Red Packet（创建红包）、Claim Page（领取页面）、My Packets（我的红包记录）、Tip Button（嵌入式打赏按钮）
- 后端生成短链接和二维码，追踪红包状态

## 技术架构

创建者通过 PTB 将资金锁定到 RedPacket 对象中，设置数量和模式。合约生成唯一 ID，后端将其编码为短链接或二维码。领取者点击链接后前端构建 PTB：验证红包未过期和未领完 → 计算领取金额（随机或等额）→ 转账到领取者地址 → 更新红包状态。过期后剩余资金退还创建者。打赏功能类似，但资金直接从打赏者转入创作者地址，无锁定环节。

|------|------|------|
