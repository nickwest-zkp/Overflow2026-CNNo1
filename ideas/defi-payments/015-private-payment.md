# 隐私支付通道 / Private Payment Channel

> 基于 Seal 构建的消费级隐私支付，交易金额和对手方不公开

## 解决什么问题

区块链上所有交易公开透明，任何人都可以查看支付金额和对手方信息。这对消费级支付场景（如工资发放、B2B 交易）是重大隐私问题。企业不希望竞争对手知道供应商付款金额，个人不希望他人知道消费记录。隐私支付通道基于 Sui 的 Seal 加密框架构建，让交易金额和对手方信息只有参与方可见。

## 核心功能

- 加密交易金额：交易金额通过 Seal 加密，只有发送方和接收方可解密
- 隐蔽对手方：交易对手方地址通过零知识证明隐藏
- 隐私余额查询：用户可查询自己的隐私余额，外部无法探查
- 合规审计通道：授权审计员可查看指定范围的交易记录

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Seal 加密框架 | 加密交易金额和对手方信息 |
| Move 对象模型 | PrivatePayment 作为加密对象，数据受 Seal 保护 |
| PTB | 原子化执行"加密金额 → 验证 ZK 证明 → 执行转账" |
| 动态字段 | 存储加密的交易元数据 |
| 对象所有权 | 隐私余额对象仅所有者可读 |

## 实现方案

### 智能合约 (Move)
- `private_channel` 模块：创建隐私支付通道，管理参与方
- `encrypted_payment` 模块：处理加密金额的转账逻辑
- `zk_verifier` 模块：验证零知识证明，确认交易合法性
- `audit_gateway` 模块：授权审计员的合规查看接口
- 关键数据结构：`PrivatePayment { encrypted_amount: vector<u8>, zk_proof: vector<u8>, sender_commitment: vector<u8>, recipient_commitment: vector<u8> }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + Seal SDK + ZK 库
- 关键页面：Private Dashboard（隐私余额总览）、Send Private（发起隐私支付）、Channel Management（通道管理）、Audit View（审计视图）
- 后端提供 ZK 证明生成辅助服务

## 技术架构

用户创建隐私支付通道时，系统通过 Seal 生成加密密钥对。发起隐私支付时，前端在本地加密金额和对手方信息，生成 ZK 证明证明交易合法性（余额充足），通过 PTB 提交到合约。合约验证 ZK 证明后执行内部账本更新，不暴露明文数据。接收方通过 Seal 解密查看金额。授权审计员可通过 audit_gateway 模块查看指定范围交易记录。所有加密数据存储在链上动态字段中。
