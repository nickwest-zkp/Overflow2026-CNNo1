# 社交支付 / Social Payment

> 通过社交关系（ENS、链上好友）发送付款，无需输入地址

## 解决什么问题

加密货币转账需要输入一长串十六进制地址，用户体验极差。普通用户难以记住和验证地址是否正确，输错地址则资金永久丢失。社交支付让用户通过社交身份（链上域名、社交图谱好友关系）而非地址来发送付款，降低使用门槛，让链上支付像微信转账一样简单。

## 核心功能

- 身份解析：输入 SuiNS 域名、ENS 或社交账号即可找到对应的收款地址
- 好友列表：基于链上交互频率自动生成好友推荐列表
- 一键转账：从好友列表选择联系人，输入金额即可完成转账
- 社交动态：转账附带留言和表情，生成链上社交记忆

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| PTB | 原子化执行"解析身份 → 验证地址 → 执行转账 → 发送通知" |
| Coin 标准 | 转账使用 Coin<T> |
| SuiNS 集成 | 解析域名到地址 |
| 动态字段 | 存储用户社交资料和好友关系 |
| 对象所有权 | UserProfile 对象绑定用户身份 |

## 实现方案

### 智能合约 (Move)
- `social_registry` 模块：注册用户社交身份，关联钱包地址
- `friend_graph` 模块：管理链上好友关系，支持添加和移除
- `social_transfer` 模块：处理带留言的社交转账
- `identity_resolver` 模块：集成 SuiNS 和其他域名系统
- 关键数据结构：`SocialProfile { address: address, name: String, friends: vector<address>, transfer_count: u64, created_at: u64 }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + SuiNS SDK
- 关键页面：Contacts（联系人）、Send Payment（发送支付）、Activity Feed（社交动态流）、Profile（个人资料）
- 后端索引链上交易构建社交图谱和好友推荐

## 技术架构

用户首次使用时通过 PTB 创建 SocialProfile 对象，设置链上昵称和头像。后端索引器扫描用户的历史交易记录，基于交互频率生成好友推荐列表。发送转账时，用户输入好友名称或域名，前端通过 SuiNS 解析地址，构建 PTB 执行转账并附带留言。转账记录展示在社交动态流中，好友可以点赞或评论。所有社交数据存储在链上动态字段中。
