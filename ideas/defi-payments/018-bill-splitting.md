# 账单分摊应用 / Bill Splitting App

> 多人聚餐/旅行后链上分摊账单，自动计算各应付金额，一键支付

## 解决什么问题

多人聚餐、合租或旅行后的账单分摊总是让人头疼。传统方式需要一人先垫付，其他人通过微信或银行转账还款，操作繁琐且容易遗漏。加密货币用户更缺乏便捷的分摊工具。账单分摊应用让创建者上传账单，系统自动计算每人应付金额，参与者通过 PTB 一键完成链上支付，所有记录在链上可追溯。

## 核心功能

- 智能账单创建：手动输入或拍照识别账单，添加参与者和分摊规则
- 灵活分摊模式：支持等额分摊、按比例分摊、指定金额分摊
- 一键支付：每位参与者通过 PTB 一键支付自己应付的份额
- 债务结算优化：多笔账单后自动抵消债务，减少实际转账次数

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | BillSplit 作为独立 Object，存储账单和支付状态 |
| PTB | 原子化执行"支付份额 → 更新状态 → 通知创建者" |
| Coin 标准 | 支持任意 Coin<T> 作为支付代币 |
| 动态字段 | 存储参与者列表、应付金额和支付记录 |
| 事件 (Events) | 发出支付完成事件，触发通知和债务更新 |

## 实现方案

### 智能合约 (Move)
- `bill_factory` 模块：创建分摊账单，定义总金额和参与者
- `split_engine` 模块：按分摊规则计算每人应付金额
- `payment_tracker` 模块：追踪每位参与者的支付状态
- `debt_optimizer` 模块：多笔账单间的债务抵消和最优化结算
- 关键数据结构：`BillSplit { creator: address, total: Coin<T>, splits: vector<SplitItem>, settled: bool, deadline: u64 }`

### 前端 / 后端
- 技术栈：React Native + @mysten/dapp-kit + OCR SDK
- 关键页面：Create Bill（创建账单）、Split Detail（分摊详情）、Pay My Share（支付份额）、Debt Summary（债务总览）
- 后端提供 OCR 账单识别和债务优化计算

## 技术架构

创建者通过 App 创建账单，输入总金额、添加参与者、选择分摊模式。合约创建 BillSplit 对象并通知参与者。每位参与者查看账单详情后，通过 PTB 支付自己的份额到合约锁定。当所有份额收齐后，合约将总金额转给创建者并标记为已结算。如果有参与者超时未付，创建者可发起催付或调整分摊。多笔账单后，debt_optimizer 计算最优结算路径，减少链上交易次数。
