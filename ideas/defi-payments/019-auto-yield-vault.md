# 自动收益金库 / Auto Yield Vault

> 用户存入资产，策略自动在 Sui DeFi 协议间切换，追求最优收益

## 解决什么问题

Sui 生态中有多个 DeFi 协议提供不同收益率的理财产品，用户需要在各协议间手动比较和切换，耗费大量时间且容易错过最佳收益机会。APY 随市场条件实时变化，手动追踪几乎不可能。自动收益金库让用户存入资产后，策略引擎自动监控各协议收益率并在最优协议间切换，最大化用户收益。

## 核心功能

- 多协议收益聚合：集成 Scallop、Cetus、DeepBook 等 Sui DeFi 协议
- 自动策略切换：当某协议 APY 下降时自动将资金转移到更高收益的协议
- 复利自动再投资：收益自动复投，无需手动操作
- 风险评级系统：为每个策略标注风险等级，用户可选择风险偏好

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | YieldVault 作为共享对象，管理用户存款和策略状态 |
| PTB | 原子化执行"提取 → 兑换 → 存入新协议"策略切换 |
| Coin 标准 | 统一处理多种资产类型 |
| 动态字段 | 存储用户份额、策略配置和收益记录 |
| 共享对象 | 金库资金池允许多用户并发存取 |

## 实现方案

### 智能合约 (Move)
- `vault_factory` 模块：创建收益金库，定义支持的资产类型
- `strategy_registry` 模块：注册和管理可用策略（各协议适配器）
- `rebalance_engine` 模块：执行策略切换和资金再分配
- `share_token` 模块：铸造和销毁金库份额代币
- 关键数据结构：`YieldVault { asset_type: Type, total_deposits: Coin<T>, active_strategy: Strategy, share_supply: u64, last_rebalance: u64 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Recharts
- 关键页面：Vault Dashboard（金库总览）、Deposit/Withdraw（存取款）、Strategy View（当前策略）、Performance（收益曲线）
- 后端 Keeper 监控 APY 变化并触发再平衡

## 技术架构

用户通过 PTB 存入资产到 YieldVault 共享对象，合约铸造等比例份额代币给用户。后端 Keeper 持续监控已注册协议的 APY 数据，当发现更优策略时通过 PTB 触发再平衡：从当前协议提取资金 → 通过 DEX 兑换（如需）→ 存入新协议。收益自动累积并定期复投，用户的份额代币价值随之增长。用户可随时通过 PTB 销毁份额代币提取资产。
