# 目标储蓄器 / Goal-Based Savings

> 设定储蓄目标（如"3 个月内存 1000 USDC"），系统自动定期扣款并投资

## 解决什么问题

大多数人缺乏系统化的储蓄习惯，传统的银行储蓄利率低且缺乏激励。加密货币用户虽然持有资产，但缺乏将储蓄目标与投资收益结合的工具。目标储蓄器让用户设定具体的储蓄目标，系统自动从用户钱包定期扣款，扣款金额自动投入收益策略，让储蓄过程既有纪律性又有收益增长。

## 核心功能

- 可视化储蓄目标：创建带名称、金额、截止日期的储蓄目标，进度条实时展示
- 自动定期扣款：按设定频率（日/周/月）自动从用户钱包转入目标账户
- 收益加速：存入资金自动投入低风险 DeFi 策略加速目标达成
- 社交激励：朋友可为你的储蓄目标助力打赏，达成目标时铸造成就 NFT

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | SavingsGoal 作为独立 Object，存储目标和资金 |
| PTB | 原子化执行"扣款 → 投入策略 → 更新进度" |
| Coin 标准 | 储蓄资金使用 Coin<T> |
| NFT | 达成目标时铸造成就纪念 NFT |
| 动态字段 | 存储扣款计划、进度记录和助力记录 |

## 实现方案

### 智能合约 (Move)
- `goal_factory` 模块：创建储蓄目标，设置金额和截止日期
- `auto_deposit` 模块：管理定期扣款计划和授权
- `yield_strategy` 模块：将存款投入低风险 DeFi 协议赚取收益
- `achievement` 模块：目标达成时铸造成就 NFT 和社交分享
- 关键数据结构：`SavingsGoal { owner: address, target: Coin<T>, current: Coin<T>, deadline: u64, deposit_schedule: Schedule, status: u8 }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：My Goals（我的目标）、Create Goal（创建目标）、Goal Detail（目标详情）、Community（社区激励）
- 后端 Keeper 按计划触发扣款，计算目标预计达成时间

## 技术架构

用户创建储蓄目标时通过 PTB 授权合约定期扣款，合约创建 SavingsGoal 对象。后端 Keeper 按设定频率通过 PTB 执行扣款：从用户钱包提取指定金额 → 存入目标对象 → 自动投入收益策略。前端实时计算并展示进度百分比和预计达成时间（考虑收益加速）。当累计金额达到目标时，合约自动铸造成就 NFT，用户可选择提取资金或继续储蓄。朋友可通过 PTB 为目标助力打赏。
