# 指数投资协议 / Index Investing Protocol

> 创建链上指数代币（如 Sui Top 10），自动调仓，一键买入一篮子资产

## 解决什么问题

投资者希望分散投资 Sui 生态的多个代币，但手动购买和管理多个资产既费时又容易配比不均。传统指数基金由金融机构管理，准入门槛高且不透明。指数投资协议让任何人都能创建或投资链上指数代币，一键买入一篮子资产，合约自动根据市值权重调仓，完全去中心化和透明。

## 核心功能

- 指数创建：自定义指数成分（如 Sui Top 10 代币）和权重计算方式
- 一键买入/赎回：存入一种代币，通过 PTB 自动分配购买所有成分代币
- 自动调仓：定期根据市值变化自动调整各成分权重
- 指数代币流通：指数份额铸造为可转让代币，可在 DEX 上自由交易

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | IndexFund 作为共享对象，管理成分资产和权重 |
| PTB | 原子化执行"存入 → 拆分 → 逐个购买成分代币 → 铸造指数代币" |
| Coin 标准 | 指数代币和成分代币都使用 Coin<T> |
| 动态字段 | 存储成分列表、目标权重、当前权重和调仓历史 |
| 共享对象 | 指数基金池允许多用户并发申购赎回 |

## 实现方案

### 智能合约 (Move)
- `index_factory` 模块：创建新指数，定义成分和权重规则
- `mint_redeem` 模块：处理指数代币的铸造（申购）和销毁（赎回）
- `rebalance` 模块：定期调仓，按目标权重调整各成分持有量
- `weight_calculator` 模块：根据市值、等权等规则计算目标权重
- 关键数据结构：`IndexFund { components: vector<ComponentToken>, target_weights: vector<u64>, index_token_supply: u64, last_rebalance: u64 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Recharts
- 关键页面：Index Market（指数市场）、Create Index（创建指数）、My Holdings（我的持仓）、Index Detail（指数详情和成分）
- 后端提供市值数据和调仓触发服务

## 技术架构

指数创建者通过合约定义成分代币和权重规则（如按市值加权）。投资者存入 USDC 等基础代币，合约通过 PTB 原子化执行：将存入金额按权重拆分 → 通过 DeepBook 逐个购买成分代币 → 铸造等比例指数代币给投资者。赎回时反向操作：销毁指数代币 → 卖出所有成分代币 → 将基础代币退还。后端定期触发调仓，根据最新市值数据调整权重并执行买卖操作。
