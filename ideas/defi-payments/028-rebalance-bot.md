# 再平衡机器人 / Rebalance Bot

> 资产偏离目标配比时自动调整，减少滑点，优化 Gas

## 解决什么问题

投资组合需要定期再平衡以维持目标资产配比，但手动操作需要持续监控市场价格并计算偏离程度。频繁的再平衡产生高额 Gas 费和交易滑点，不频繁再平衡则组合偏离目标过远。再平衡机器人根据偏离阈值智能触发再平衡，通过批量交易和最优路径规划减少滑点和 Gas 成本。

## 核心功能

- 偏离检测：持续监控组合中各资产的权重偏离程度
- 智能触发：只在偏离超过阈值时触发再平衡，避免不必要的交易
- 最优路径规划：计算最少的交易步骤完成再平衡，减少滑点
- Gas 优化：将多笔交易合并为一个 PTB 批量执行，节省 Gas

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | RebalancePlan 作为独立 Object，存储目标配比和当前持仓 |
| PTB | 将多笔调整交易合并为一个 PTB 原子化执行 |
| Coin 标准 | 处理多种资产的转换 |
| 动态字段 | 存储目标权重、当前权重和再平衡历史 |
| DeepBook 集成 | 通过 DEX 执行资产兑换 |

## 实现方案

### 智能合约 (Move)
- `portfolio_manager` 模块：创建投资组合并设定目标配比
- `deviation_checker` 模块：计算当前权重与目标权重的偏离度
- `rebalance_executor` 模块：构建并执行再平衡 PTB
- `gas_optimizer` 模块：计算最少交易步骤的最优路径
- 关键数据结构：`Portfolio { owner: address, holdings: vector<Coin<T>>, target_weights: vector<u64>, tolerance: u64, last_rebalance: u64 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Portfolio Setup（组合配置）、Deviation Monitor（偏离监控）、Rebalance History（再平衡记录）、Gas Savings（Gas 节省统计）
- 后端监控偏离并触发再平衡

## 技术架构

用户创建组合时定义各资产的目标权重和再平衡容差（如偏离 >5% 时触发）。后端持续追踪各资产的市值变化并计算当前权重。当偏离超过容差时，后端计算最优再平衡路径（最少交易步骤），构建 PTB 原子化执行：卖出超配资产 → 买入低配资产 → 更新持仓记录。多笔调整合并为一个 PTB，减少 Gas 和滑点。前端展示偏离趋势和再平衡效果。
