# 清算机器人 / Liquidation Bot

> 监控借贷协议，发现可清算仓位时自动执行，赚取清算奖励

## 解决什么问题

DeFi 借贷协议依赖清算人来维持系统健康，但清算机会竞争激烈，需要极快的反应速度和精确的 Gas 竞价策略。手动清算效率低下且容易错过机会。清算机器人 24/7 监控所有借贷协议的仓位健康状态，发现可清算仓位时立即通过 PTB 执行清算，最大化清算奖励收益。

## 核心功能

- 全协议仓位监控：监控 Sui 上所有借贷协议的仓位健康因子
- 智能清算优先级：根据清算奖励大小和竞争程度排列优先级
- Gas 竞价策略：根据竞争情况动态调整 Gas 价格确保清算成功
- 清算收益优化：选择最有利的清算方式（部分清算 vs 全额清算）

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| PTB | 原子化执行"检查健康因子 → 执行清算 → 获得清算奖励" |
| Coin 标准 | 处理清算涉及的多种代币 |
| 共享对象 | 借贷池作为共享对象，支持并发清算 |
| 动态字段 | 存储清算历史、收益记录和 Gas 策略参数 |
| 预言机集成 | 通过 Pyth 获取实时价格计算健康因子 |

## 实现方案

### 智能合约 (Move)
- `liquidation_executor` 模块：执行清算操作，从借贷协议获取折扣资产
- `health_monitor` 模块：计算所有仓位的健康因子并排序
- `gas_strategy` 模块：根据竞争情况动态调整 Gas 竞价
- `profit_tracker` 模块：记录清算收益和 Gas 消耗
- 关键数据结构：`LiquidationOpportunity { position_id: ID, health_factor: u64, collateral_value: u64, discount: u64, estimated_profit: u64 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + WebSocket
- 关键页面：Liquidation Dashboard（清算总览）、Position Monitor（仓位监控）、Execution History（执行历史）、Profit Report（收益报告）
- 后端高频监控仓位状态并触发清算

## 技术架构

后端监控服务持续扫描所有借贷协议的仓位数据，通过预言机价格计算每个仓位的健康因子。当发现健康因子低于清算阈值的仓位时，按清算奖励大小排序。构建 PTB 执行清算：借入资金 → 偿还目标仓位债务 → 获得折扣抵押品 → 在 DEX 卖出抵押品获利。整个过程原子化完成，如果任何步骤失败则回滚。前端展示清算机会、执行记录和累计收益。
