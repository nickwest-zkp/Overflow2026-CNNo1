# AI Report Generator / AI 报告生成器

> 输入主题后 Agent 自动研究、分析、生成完整报告（PDF/Markdown），所有版本持久化存储在 Walrus

## 解决什么问题

生成一份高质量的行业分析报告通常需要数天的调研和撰写时间。现有 AI 工具生成的报告缺乏可追溯性和版本管理，且无法跨会话持续改进。本方案让 AI Agent 自动完成从调研到成稿的全流程，利用 Walrus 存储所有中间产物和最终报告，确保研究过程透明可审计。

## 核心功能

- 一键主题研究：用户输入主题关键词，Agent 自动搜集资料、提取关键信息
- 多章节自动生成：Agent 按报告结构（摘要、背景、分析、结论）逐步生成内容
- 版本迭代管理：每次修改生成新版本，支持对比任意两个版本的差异
- 引用溯源：报告中每个数据点关联原始来源 Blob，可一键查看原始材料

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储原始研究材料、中间草稿、最终报告（Markdown/PDF） |
| Sui 对象模型 | 链上记录报告元信息、作者、版本链、引用关系 |
| MemWal 记忆层 | 维护报告生成过程中 Agent 的写作风格偏好和上下文 |
| Walrus 读取 API | 按版本号获取历史报告，支持内容回溯 |

## 实现方案

### Agent / AI 部分
- 采用 LangGraph 构建多步骤报告生成流水线：调研 → 大纲 → 各章节撰写 → 整合润色
- 使用 GPT-4o 进行深度内容生成，Claude 进行事实核查
- 每个步骤独立运行，中间结果暂存为 Blob，失败可从断点恢复
- 支持 Agent 根据用户反馈迭代修改特定章节

### Walrus 集成
- 原始材料以 `reports/{id}/sources/` 目录结构存储
- 各章节草稿以 `reports/{id}/drafts/ch{N}_v{V}.md` 存储
- 最终报告以 `reports/{id}/final_v{V}.md` 和 `.pdf` 双格式存储
- 元数据对象包含：title、topic、created_at、version_chain、blob_refs

### 前端 / 后端
- 前端：Next.js + Rich Text Editor（支持 Markdown 实时预览）
- 后端：FastAPI + Celery 异步任务队列处理长时间报告生成
- 关键页面：报告创建页、生成进度页、报告阅读页、版本对比页

## 技术架构

用户输入主题 → 后端创建报告任务 → Agent 调研并收集原始材料存入 Walrus → Agent 按大纲逐章生成内容 → 中间草稿存入 Walrus → 最终整合润色 → 完整报告 PDF/Markdown 存入 Walrus → Sui 链上记录报告对象和版本链 → 前端渲染报告并支持版本切换。
