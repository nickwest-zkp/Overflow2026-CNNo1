# Dataset Builder Agent / 数据集构建智能体

> Agent 从多个源自动收集数据，经清洗、标注、格式化后存储在 Walrus，构建可复用的去中心化数据集

## 解决什么问题

高质量数据集是 AI 训练的基石，但构建过程繁琐且缺乏标准化。现有数据集通常托管在中心化平台，存在不可用或被篡改的风险。本方案让 AI Agent 自动化数据收集和预处理全流程，利用 Walrus 确保数据集持久可用、不可篡改、可追溯来源。

## 核心功能

- 多源数据采集：Agent 从 API、网页、链上数据等多个源自动收集原始数据
- 自动清洗标注：AI 自动去重、纠错、标准化格式，并添加分类标签
- 数据集发布：处理后的数据集存储在 Walrus 并在链上注册，其他项目可引用
- 更新追踪：数据集可设置定期更新策略，自动采集新数据并发布新版本

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储原始数据、清洗后数据集、Schema 定义文件 |
| Sui 对象模型 | 链上注册数据集元信息、版本、许可证、引用计数 |
| Seal 加密 | 对含隐私数据的数据集进行加密，授权访问后可解密 |
| Walrus 读取 API | 支持其他项目按版本号读取和下载数据集 |

## 实现方案

### Agent / AI 部分
- 采集 Agent 使用 Playwright 爬取网页数据，通过 API 获取结构化数据
- 清洗 Agent 使用 LLM 识别异常数据、去重、统一格式
- 标注 Agent 自动为数据添加分类标签和元数据描述
- 质检 Agent 抽样检查数据质量，生成质量报告

### Walrus 集成
- 原始数据以 `datasets/{id}/raw/part-{N}.jsonl` 分片存储
- 清洗后数据以 `datasets/{id}/clean/v{V}.jsonl` 整体存储
- Schema 定义以 `datasets/{id}/schema.json` 存储，描述字段含义和类型
- Sui 链上对象记录数据集名称、描述、大小、Blob 列表、更新频率

### 前端 / 后端
- 前端：Vue.js + ECharts，展示数据集浏览、质量指标、下载统计
- 后端：Python FastAPI，提供数据集 CRUD 和采集任务管理 API
- 关键页面：数据集市场页、数据集详情页、构建进度页、质量报告页

## 技术架构

用户定义数据集目标和采集规则 → 采集 Agent 按规则从多源收集数据 → 原始数据存入 Walrus → 清洗 Agent 处理数据 → 标注 Agent 添加标签 → 质检 Agent 验证 → 最终数据集存入 Walrus → Sui 链上注册数据集对象 → 其他项目通过链上索引发现并引用数据集。

## 难度评估

| 维度 | 难度 | 说明 |
|------|------|------|
| Walrus 集成 | ⭐⭐⭐ | 大文件分片存储、版本管理、跨项目引用 |
| AI/Agent | ⭐⭐⭐ | 多步骤数据清洗、自动标注和质量控制 |
| 前端 | ⭐⭐ | 数据集浏览和统计展示 |
| 集成复杂度 | ⭐⭐⭐ | 多 Agent 流水线 + 数据分片 + 链上注册 |
