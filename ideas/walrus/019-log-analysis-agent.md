# Log Analysis Agent / 链上日志分析智能体

> 持续收集 Sui 链上事件日志，AI 自动分析异常模式，分析报告和原始日志持久化在 Walrus

## 解决什么问题

链上活动日益复杂，手动监控大量事件日志几乎不可能。现有的链上分析工具通常只提供基础查询，缺乏智能异常检测和长期趋势分析能力。本方案通过 AI Agent 7x24 小时监控链上事件，利用 Walrus 存储完整的日志历史和分析报告，让安全问题不会被遗漏。

## 核心功能

- 实时日志收集：通过 Sui RPC 持续订阅链上事件，按时间窗口归档到 Walrus
- 异常模式检测：AI 分析日志中的异常交易模式（大额转账、异常合约调用、闪电贷攻击）
- 趋势分析报告：每日/每周自动生成链上活动趋势报告，包含关键指标变化
- 事件回溯查询：用户可查询任意历史时间段的事件日志和分析结果

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储原始日志（JSON Lines）、分析报告、异常事件索引 |
| Sui 事件订阅 | 实时获取链上事件作为数据源 |
| Sui 对象模型 | 链上记录日志归档元信息、分析结果摘要、异常告警状态 |
| MemWal 记忆层 | Agent 记住已识别的异常模式基线，跨会话持续优化检测模型 |

## 实现方案

### Agent / AI 部分
- 日志收集模块通过 Sui WebSocket 订阅事件，按 1 小时窗口归档
- 异常检测 Agent 使用统计模型 + LLM 语义分析双通道检测异常
- 趋势分析 Agent 每日汇总关键指标（交易量、活跃地址、Gas 消耗等）
- 告警 Agent 对高优先级异常生成实时通知（邮件/Webhook/Telegram）

### Walrus 集成
- 原始日志按小时存储：`logs/{date}/{hour}.jsonl`
- 每日分析报告：`reports/daily/{date}.md`
- 异常事件索引：`alerts/{date}.json`，包含异常类型、严重程度、相关交易
- 基线数据：`baselines/{metric}.json`，记录各指标的正常范围

### 前端 / 后端
- 前端：React + D3.js，展示实时日志流、异常告警列表、趋势图表
- 后端：Go 微服务处理高吞吐日志收集，Python 服务运行 AI 分析
- 关键页面：实时监控仪表盘、告警列表页、趋势分析页、日志搜索页

## 技术架构

Sui 链上事件 → WebSocket 实时推送到日志收集服务 → 按 1 小时窗口写入 Walrus Blob → 异常检测 Agent 实时分析新日志 → 发现异常时生成告警 → 告警记录存入 Walrus → 每日趋势分析 Agent 读取当日所有日志 → 生成分析报告存入 Walrus → 前端从 Walrus 读取并渲染图表和告警。
