# Auto Doc Maintainer / 文档自动维护智能体

> Agent 监控代码仓库变更，自动更新对应文档，所有文档版本存储在 Walrus 形成完整变更历史

## 解决什么问题

软件项目文档经常与代码不同步，开发者往往忽视文档更新。过时的文档比没有文档更危险，会导致新成员困惑和 API 误用。本方案通过 AI Agent 持续监控代码变更，自动识别需要更新的文档并生成修改，利用 Walrus 存储所有文档版本确保变更历史永不丢失。

## 核心功能

- 代码变更监控：通过 GitHub Webhook 实时监听代码仓库的提交和 PR
- 文档影响分析：AI 分析代码变更影响范围，确定哪些文档需要更新
- 自动文档更新：根据代码 diff 生成文档修改建议，支持自动合并或人工审核
- 版本历史管理：每次文档变更作为新版本存入 Walrus，支持任意版本回溯

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储文档 Markdown 文件各版本、变更摘要、代码 diff |
| Sui 对象模型 | 链上记录仓库元信息、文档版本链、审核状态 |
| MemWal 记忆层 | 记住项目的文档风格规范、常用术语、架构决策 |
| Walrus 读取 API | 获取历史文档版本，支持版本对比 |

## 实现方案

### Agent / AI 部分
- 监控 Agent 通过 GitHub API 监听目标仓库的 push 和 PR 事件
- 分析 Agent 使用 Tree-sitter 解析代码变更，识别影响到的模块和接口
- 文档生成 Agent 基于代码 diff 和现有文档生成更新内容
- 审核流程：自动通过低风险修改，高风险修改生成 PR 等待人工审核

### Walrus 集成
- 文档各版本以 `docs/{repo}/{path}/v{N}.md` 存储
- 变更记录以 `docs/{repo}/changes/{commit_sha}.json` 存储
- 文档索引以 `docs/{repo}/index.json` 存储，记录所有文档路径和最新版本
- 仓库配置以 `docs/{repo}/config.json` 存储，包含监控规则和文档风格

### 前端 / 后端
- 前端：React + Monaco Editor，展示文档对比和在线编辑
- 后端：Node.js 处理 Webhook，Python 运行 AI 分析
- 关键页面：仓库仪表盘、文档变更列表、版本对比页、审核队列

## 技术架构

GitHub 仓库推送代码 → Webhook 触发后端 → 监控 Agent 获取代码 diff → 分析 Agent 识别受影响文档 → 文档生成 Agent 生成更新内容 → 更新内容存入 Walrus → 低风险自动合并、高风险生成 PR → 文档版本链更新 → 前端展示变更历史。
