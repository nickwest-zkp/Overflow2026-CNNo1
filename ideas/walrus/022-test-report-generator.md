# Test Report Generator / 测试报告生成器

> Agent 自动运行测试套件并分析结果，生成人类可读的测试报告存储在 Walrus，追踪质量趋势

## 解决什么问题

测试结果通常以机器格式输出，开发者难以快速理解失败原因。跨版本的测试质量趋势缺乏可视化追踪，测试覆盖率的变化往往被忽视。本方案让 AI Agent 自动执行测试、分析结果、生成可读报告，所有历史报告存储在 Walrus 确保质量数据永不丢失。

## 核心功能

- 自动测试执行：集成 CI/CD 流水线，每次代码提交自动触发测试
- 智能结果分析：AI 分析失败测试的根本原因，生成自然语言解释
- 趋势追踪：追踪测试通过率、覆盖率、执行时间等指标的变化趋势
- 失败模式识别：识别反复出现的失败模式，提示潜在系统性问题

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储原始测试输出、分析报告、趋势数据 |
| Sui 对象模型 | 链上记录项目信息、测试运行 ID、质量指标摘要 |
| MemWal 记忆层 | 记住项目的测试基线数据和历史失败模式 |
| Walrus 读取 API | 获取历史测试报告进行趋势分析 |

## 实现方案

### Agent / AI 部分
- 测试执行 Agent 调用项目的测试框架（Jest/Pytest/Cargo test）
- 结果分析 Agent 解析测试输出，使用 LLM 为每个失败测试生成人类可读解释
- 趋势分析 Agent 读取 MemWal 中历史数据，计算指标变化并生成趋势图
- 根因分析 Agent 对关联的失败测试进行聚类分析，识别系统性问题

### Walrus 集成
- 原始测试输出以 `tests/{project}/{run_id}/raw.json` 存储
- 分析报告以 `tests/{project}/{run_id}/report.md` 存储
- 趋势数据以 `tests/{project}/trends.json` 累积更新
- 失败模式库以 `tests/{project}/patterns.json` 存储

### 前端 / 后端
- 前端：React + Recharts，展示测试结果和趋势图表
- 后端：Node.js 提供 CI/CD Webhook 接收和报告查询 API
- 关键页面：测试仪表盘、单次测试报告页、趋势分析页、失败模式页

## 技术架构

代码提交触发 CI/CD → Webhook 通知后端 → 测试执行 Agent 运行测试套件 → 原始输出存入 Walrus → 结果分析 Agent 解析并生成报告 → 报告存入 Walrus → 趋势分析 Agent 更新趋势数据 → 前端从 Walrus 读取并渲染报告和趋势图。
