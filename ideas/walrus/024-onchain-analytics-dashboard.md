# Onchain Analytics Dashboard / 链上数据分析仪表盘

> Agent 定期生成链上分析数据集存入 Walrus，前端读取并渲染为交互式可视化图表

## 解决什么问题

Sui 链上数据丰富但缺乏直观的可视化分析工具。现有区块浏览器只提供基础查询，无法进行趋势分析和模式发现。本方案通过 AI Agent 持续采集和分析链上数据，生成结构化分析数据集存储在 Walrus，前端提供丰富的交互式图表帮助用户洞察链上活动。

## 核心功能

- 自动数据采集：定期从 Sui RPC 采集交易、事件、对象变更等数据
- 多维度分析：按地址、时间、DApp、交易类型等维度聚合分析
- 可视化图表：折线图、热力图、桑基图等多种图表类型
- 自定义看板：用户可创建自定义仪表盘，选择关注的数据维度

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储分析数据集（Parquet/JSON）、图表配置、用户看板 |
| Sui RPC | 获取链上交易、事件、检查点等原始数据 |
| Sui 对象模型 | 链上记录数据集版本、分析任务状态 |
| MemWal 记忆层 | 存储用户自定义看板配置和分析偏好 |

## 实现方案

### Agent / AI 部分
- 采集 Agent 通过 Sui RPC 分页获取每日交易和事件数据
- 聚合 Agent 按多个维度（时间/地址/DApp/类型）计算统计指标
- 异常检测 Agent 识别数据中的异常峰值和模式变化
- 洞察生成 Agent 自动生成数据洞察摘要（如「今日活跃地址增长 15%」）

### Walrus 集成
- 原始数据快照以 `analytics/raw/{date}.jsonl` 存储
- 聚合数据集以 `analytics/agg/{dimension}/{date}.json` 存储
- 图表配置以 `dashboards/{user_id}/config.json` 存储
- 洞察摘要以 `analytics/insights/{date}.md` 存储

### 前端 / 后端
- 前端：React + Apache ECharts，支持拖拽式看板自定义
- 后端：Python FastAPI + Apache Arrow 高效处理分析数据
- 关键页面：总览仪表盘、交易分析页、地址画像页、自定义看板页

## 技术架构

定时任务触发采集 Agent → 从 Sui RPC 获取当日数据 → 原始数据存入 Walrus → 聚合 Agent 计算多维度统计 → 聚合数据集存入 Walrus → 异常检测 Agent 识别异常 → 洞察生成 Agent 生成摘要 → 前端从 Walrus 读取数据集 → ECharts 渲染交互式图表。
