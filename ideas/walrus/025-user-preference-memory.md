# User Preference Memory / 用户偏好记忆层

> Agent 自动记住用户偏好（风险等级、兴趣赛道、语言），跨会话提供个性化去中心化服务

## 解决什么问题

每次与 AI Agent 对话都需要重新说明个人偏好，用户体验极差。传统 AI 助手的偏好存储在中心化服务器上，用户无法掌控自己的数据。本方案将用户偏好持久化在 Walrus，通过 Seal 加密保护隐私，用户完全掌控自己的偏好数据，且可在不同 Agent 间共享。

## 核心功能

- 偏好自动学习：Agent 从用户对话和行为中自动提取偏好特征（语言、风格、专业领域）
- 跨会话记忆：新会话启动时自动加载用户偏好，提供个性化体验
- 隐私保护：偏好数据使用 Seal 加密，仅用户授权的 Agent 可解密
- 偏好管理界面：用户可查看、编辑、删除自己的偏好数据

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储用户偏好 JSON 文件、行为历史摘要 |
| Seal 加密 | 加密偏好数据，仅授权 Agent 可访问 |
| Sui 对象模型 | 链上记录用户偏好 Blob 引用、授权列表 |
| MemWal 记忆层 | 管理 Agent 的短期会话上下文和长期偏好关联 |

## 实现方案

### Agent / AI 部分
- 偏好提取 Agent 使用 NLU 从对话中识别用户偏好（如「我比较保守」→ 风险厌恶）
- 偏好更新 Agent 在每次对话结束时计算偏好变化并更新
- 个性化推荐 Agent 基于偏好提供定制化内容和建议
- 偏好推理 Agent 推断隐含偏好（如用户总查询 DeFi → 对 DeFi 感兴趣）

### Walrus 集成
- 用户偏好文件以 `preferences/{user_id}/profile.json` 存储
- 行为摘要以 `preferences/{user_id}/behavior_summary.json` 存储
- 授权列表存储在 Sui 链上对象中，记录哪些 Agent 有权访问
- 偏好变更历史以 `preferences/{user_id}/history/{date}.json` 追踪

### 前端 / 后端
- 前端：React + Ant Design，展示偏好管理面板
- 后端：FastAPI 提供 Agent 访问偏好数据的 API（带授权验证）
- 关键页面：偏好仪表盘、偏好编辑页、授权管理页、隐私设置页

## 技术架构

用户与 Agent 对话 → 偏好提取 Agent 从对话中识别偏好 → 偏好更新 Agent 合并到已有偏好 → 新偏好使用 Seal 加密 → 加密偏好存入 Walrus Blob → Sui 链上更新偏好引用 → 下次对话时 Agent 从 Walrus 读取并解密偏好 → 提供个性化服务。
