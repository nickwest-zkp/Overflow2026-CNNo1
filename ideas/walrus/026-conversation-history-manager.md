# Conversation History Manager / 对话历史管理器

> 长期存储用户与 Agent 的完整对话记录，Agent 可引用历史对话上下文，实现真正的长期记忆

## 解决什么问题

AI 对话系统的上下文窗口有限，重要对话内容在窗口满后就被遗忘。用户无法回溯一个月前的对话细节，Agent 也无法引用历史决策的依据。本方案将完整对话历史持久化在 Walrus，Agent 可以按需检索任意历史对话，实现真正的无限记忆。

## 核心功能

- 对话自动归档：每次对话结束自动将完整消息列表存储到 Walrus
- 语义检索：Agent 根据当前话题自动检索相关历史对话片段
- 对话时间线：用户可浏览完整的对话历史时间线
- 上下文压缩：长对话自动压缩为摘要，保留关键信息减少存储占用

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储完整对话 JSON、对话摘要、语义索引 |
| Sui 对象模型 | 链上记录对话元信息（时间、主题、参与者） |
| Seal 加密 | 加密敏感对话内容，仅用户和授权 Agent 可访问 |
| MemWal 记忆层 | 管理 Agent 的短期上下文和对话摘要缓存 |

## 实现方案

### Agent / AI 部分
- 对话压缩 Agent 使用 LLM 将长对话压缩为结构化摘要（关键决策、重要信息、待办事项）
- 语义检索 Agent 使用向量嵌入将对话片段索引化，支持相似度检索
- 上下文注入 Agent 根据当前对话自动从历史中检索相关片段注入上下文
- 隐私过滤 Agent 识别并过滤对话中的敏感个人信息

### Walrus 集成
- 完整对话以 `conversations/{user_id}/{session_id}.json` 存储
- 对话摘要以 `conversations/{user_id}/summaries/{session_id}.json` 存储
- 语义索引以 `conversations/{user_id}/index/embeddings.bin` 存储
- 对话元数据在 Sui 链上记录：时间戳、主题标签、参与者

### 前端 / 后端
- 前端：React + 虚拟滚动列表，展示对话历史和搜索
- 后端：Python FastAPI，提供对话检索和语义搜索 API
- 关键页面：对话时间线页、对话搜索页、上下文查看页、隐私管理页

## 技术架构

用户与 Agent 对话 → 对话结束触发归档 → 完整对话存入 Walrus Blob → 压缩 Agent 生成摘要 → 摘要存入 Walrus → 语义索引 Agent 更新向量索引 → Sui 链上记录对话元数据 → 新对话时检索 Agent 匹配相关历史 → 注入 Agent 将历史片段加入上下文 → Agent 基于增强上下文回复。
