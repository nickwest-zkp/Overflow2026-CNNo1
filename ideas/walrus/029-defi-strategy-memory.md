# DeFi Strategy Memory / DeFi 策略记忆器

> Agent 记住用户过去执行过的 DeFi 策略和盈亏结果，避免重复犯错，持续优化投资决策

## 解决什么问题

DeFi 投资者经常重复犯同样的错误，因为没有系统化的策略复盘机制。传统工具只记录交易结果，不记录策略逻辑和市场背景。本方案通过 AI Agent 完整记录每次策略执行的上下文、参数和结果，存储在 Walrus 形成不可篡改的策略记忆库，帮助用户从历史中学习。

## 核心功能

- 策略完整记录：记录每次 DeFi 操作的策略逻辑、市场环境、执行参数和最终结果
- 盈亏分析：自动计算每个策略的 PnL、夏普比率、最大回撤等指标
- 失败模式识别：AI 识别反复出现的亏损模式，主动预警
- 策略优化建议：基于历史表现推荐参数调整和新策略

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储策略详情、执行记录、分析报告 |
| Sui 链上数据 | 读取 DeFi 协议交互记录（swap、staking、lending） |
| Sui 对象模型 | 链上记录策略摘要、PnL、风险指标 |
| Seal 加密 | 加密策略参数等敏感交易数据 |

## 实现方案

### Agent / AI 部分
- 策略记录 Agent 自动捕获用户的 DeFi 操作，关联策略上下文
- 复盘 Agent 在策略周期结束后分析执行效果，总结经验教训
- 模式识别 Agent 使用时序分析识别重复出现的亏损模式
- 优化建议 Agent 基于历史数据推荐参数调整（如调整止损点、仓位比例）
- 使用 LLM 将数据分析结果转化为可理解的策略洞察

### Walrus 集成
- 策略详情以 `strategies/{user_id}/{strategy_id}.json` 存储
- 执行记录以 `strategies/{user_id}/executions/{date}.json` 存储
- 复盘报告以 `strategies/{user_id}/reviews/{strategy_id}.md` 存储
- 模式库以 `strategies/{user_id}/patterns.json` 存储

### 前端 / 后端
- 前端：React + TradingView 图表库，展示策略回测和盈亏曲线
- 后端：Python FastAPI + pandas 数据分析
- 关键页面：策略仪表盘、策略详情页、复盘报告页、模式预警页

## 技术架构

用户执行 DeFi 操作 → 链上事件触发策略记录 → 策略记录 Agent 捕获操作详情 → 关联策略上下文存入 Walrus → 策略周期结束触发复盘 → 复盘 Agent 分析盈亏 → 报告存入 Walrus → 模式识别 Agent 更新模式库 → 优化建议 Agent 生成推荐 → 前端展示分析结果。
