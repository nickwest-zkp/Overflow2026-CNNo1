# User Profile Builder / 用户画像构建器

> 从链上行为和对话数据中构建多维度用户画像，存储在 Walrus，提供精准去中心化推荐服务

## 解决什么问题

Web3 用户数据分散在链上和各个应用中，缺乏统一的用户画像。传统推荐系统依赖中心化数据收集，用户无法控制自己的数据。本方案通过 AI Agent 自动从链上行为和对话中构建用户画像，利用 Walrus 和 Seal 实现用户自主掌控的隐私保护画像系统。

## 核心功能

- 多源数据融合：整合链上交易、DApp 交互、对话内容等多源数据
- 画像自动构建：AI 自动提取兴趣标签、行为模式、偏好特征
- 隐私保护分享：用户可选择性地向 DApp 分享画像的部分维度
- 画像持续更新：随着新数据产生自动更新画像，保持时效性

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储多维度用户画像 JSON、标签库、推荐缓存 |
| Seal 加密 | 加密画像敏感维度，选择性解密授权 |
| Sui 对象模型 | 链上记录画像版本、授权策略、分享记录 |
| Sui 链上数据 | 分析用户交易和 DApp 交互行为 |

## 实现方案

### Agent / AI 部分
- 数据采集 Agent 从多个数据源收集用户行为数据
- 特征提取 Agent 使用 NLP 和统计方法提取画像特征（兴趣、活跃度、消费力）
- 标签生成 Agent 为用户生成多层级标签（如「DeFi 爱好者 > 流动性提供者 > 稳定币偏好」）
- 推荐 Agent 基于画像生成个性化 DApp 和内容推荐
- 隐私审计 Agent 定期检查画像数据的访问合规性

### Walrus 集成
- 完整画像以 `profiles/{user_id}/full_profile.json` 加密存储
- 标签库以 `profiles/{user_id}/tags.json` 存储
- 行为摘要以 `profiles/{user_id}/behavior/{dimension}.json` 存储
- 授权策略以 Sui 链上对象形式存储，记录哪些 DApp 可访问哪些维度
- 推荐缓存以 `profiles/{user_id}/recommendations.json` 存储

### 前端 / 后端
- 前端：React + D3.js 力导向图展示画像标签关系
- 后端：Python FastAPI，提供画像查询和授权管理 API
- 关键页面：画像总览页、标签管理页、授权管理页、推荐展示页

## 技术架构

用户授权数据采集 → 采集 Agent 收集链上行为和对话数据 → 特征提取 Agent 分析数据 → 标签生成 Agent 生成多层级标签 → 完整画像使用 Seal 加密 → 加密画像存入 Walrus → 用户配置授权策略上链 → DApp 请求画像数据 → 检查链上授权 → 授权通过后解密返回对应维度。
