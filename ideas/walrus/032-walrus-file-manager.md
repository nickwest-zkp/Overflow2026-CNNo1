# Walrus File Manager / Walrus 文件管理界面

> Web UI 管理 Walrus 上的 Agent 文件，支持上传、预览、搜索和版本管理

## 解决什么问题

Walrus 作为去中心化存储平台，目前缺乏直观的文件管理界面。开发者和用户无法方便地浏览、搜索和管理存储在 Walrus 上的 Agent 数据。本方案提供一个功能完整的 Web 文件管理器，让用户像操作本地文件一样管理 Walrus 上的 Blob 数据。

## 核心功能

- 文件浏览：树形结构展示 Walrus 上的 Blob 文件，支持按目录浏览
- 在线预览：支持 Markdown、JSON、图片、PDF 等常见格式的在线预览
- 全文搜索：对所有 Blob 内容建立索引，支持全文检索
- 版本管理：查看文件的历史版本，支持版本对比和回退

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 核心 CRUD 操作：上传、读取、删除 Blob |
| Sui 对象模型 | 链上文件元数据：名称、大小、类型、创建时间 |
| Walrus 读取 API | 流式读取大文件，支持预览 |
| Sui 索引对象 | 维护目录结构和文件索引 |

## 实现方案

### Agent / AI 部分
- 智能分类 Agent 自动识别文件类型并建议分类标签
- 内容摘要 Agent 为每个文件生成简短描述，便于搜索
- 重复检测 Agent 识别内容相同但名称不同的重复文件
- 清理建议 Agent 分析存储使用模式，建议可清理的过期文件

### Walrus 集成
- 上传通过 Walrus HTTP API 直接上传文件，返回 Blob ID
- 下载通过 Walrus 读取 API 流式获取文件内容
- 目录结构维护在 Sui 链上对象中，每个目录对象包含子项列表
- 文件元数据包含：blob_id、name、size、mime_type、created_at、tags、parent_dir
- 搜索索引存储在 Walrus 上，定期由 Agent 更新

### 前端 / 后端
- 前端：React + Arco Design，类 VS Code 文件管理器界面
- 后端：Go 服务处理文件上传下载，Python 服务运行搜索
- 关键页面：文件浏览器、文件预览页、搜索结果页、存储统计页

## 技术架构

用户通过 Web UI 上传文件 → 前端分片上传到后端 → 后端调用 Walrus API 存储 Blob → Blob ID 和元数据记录到 Sui 链上对象 → 更新目录结构 → 分类 Agent 自动添加标签 → 摘要 Agent 生成文件描述 → 搜索索引更新 → 前端刷新文件列表。
