# Agent Memory Debugger / Agent 记忆调试工具

> 可视化展示 Agent 在 Walrus 中的记忆内容，支持搜索、过滤和编辑，帮助开发者调试 Agent 记忆

## 解决什么问题

AI Agent 的记忆是黑盒，开发者难以查看 Agent 到底记住了什么、为什么做出特定决策。当 Agent 行为异常时，缺乏工具来检查和调试其记忆内容。本方案提供一个专门的调试工具，可视化展示 Agent 在 Walrus 中的完整记忆结构，帮助开发者理解和优化 Agent 行为。

## 核心功能

- 记忆可视化：以思维导图或知识图谱形式展示 Agent 的记忆结构
- 内容搜索：支持按关键词、时间范围、记忆类型搜索特定记忆条目
- 记忆编辑：支持手动修改、删除、添加记忆条目进行调试
- 决策回溯：展示 Agent 每次决策引用了哪些记忆，形成决策链

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 读取和修改 Agent 的记忆数据 |
| MemWal 记忆层 | 解析 MemWal 的记忆格式和元数据 |
| Sui 对象模型 | 查询 Agent 的记忆索引和 Blob 引用列表 |
| Seal 解密 | 开发者模式下解密加密的记忆内容用于调试 |

## 实现方案

### Agent / AI 部分
- 记忆解析 Agent 将原始 Blob 数据解析为结构化的记忆图谱
- 关联分析 Agent 识别记忆条目之间的关联关系和引用链
- 异常检测 Agent 发现记忆中的矛盾、过时或冗余信息
- 优化建议 Agent 分析记忆结构效率，建议清理或重组策略
- 决策追踪 Agent 记录每次决策引用的记忆条目 ID

### Walrus 集成
- 记忆数据直接读取 Agent 存储在 Walrus 上的 Blob
- 编辑操作生成新版本 Blob，保留原始版本用于回退
- 决策日志以 `debug/{agent_id}/decisions/{session_id}.json` 存储
- 调试快照以 `debug/{agent_id}/snapshots/{timestamp}.json` 存储

### 前端 / 后端
- 前端：React + D3.js 力导向图，展示记忆关联图谱
- 后端：Python FastAPI，提供记忆查询、编辑和快照 API
- 关键页面：记忆图谱页、搜索结果页、编辑器页、决策追踪页

## 技术架构

开发者连接目标 Agent → 后端读取 Sui 链上的 Agent 索引 → 获取所有记忆 Blob ID → 从 Walrus 批量读取记忆数据 → 解析 Agent 将原始数据转为结构化图谱 → 前端渲染知识图谱 → 开发者点击节点查看详情 → 支持编辑并写回 Walrus → 决策追踪实时展示记忆引用。
