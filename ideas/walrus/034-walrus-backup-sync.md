# Walrus Backup & Sync / Walrus 备份与同步

> Agent 数据自动备份到 Walrus，支持多设备同步和版本回退，确保数据永不丢失

## 解决什么问题

AI Agent 的运行数据通常存储在本地或单点服务器上，一旦硬件故障或服务中断数据将永久丢失。多设备间 Agent 数据无法同步，用户在家中电脑和公司电脑上使用同一 Agent 时记忆不连贯。本方案提供自动化的 Walrus 备份同步服务，让 Agent 数据永远安全可恢复。

## 核心功能

- 自动增量备份：监控 Agent 数据变化，自动将增量数据备份到 Walrus
- 多设备同步：通过 Walrus 作为中转实现多设备间的数据同步
- 版本回退：保留完整版本历史，可回退到任意时间点的数据状态
- 冲突解决：多设备同时修改时自动检测冲突，提供合并策略

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储备份数据、增量快照、同步日志 |
| Sui 对象模型 | 链上记录备份版本号、设备列表、同步状态 |
| Seal 加密 | 端到端加密备份数据，仅用户设备可解密 |
| Walrus 读取 API | 其他设备从 Walrus 拉取最新备份数据 |

## 实现方案

### Agent / AI 部分
- 变更检测 Agent 监控本地 Agent 数据目录的文件变更事件
- 增量计算 Agent 对比新旧版本计算最小变更集，减少传输量
- 冲突解决 Agent 使用三向合并算法自动解决简单冲突，复杂冲突提示用户
- 同步调度 Agent 管理多设备的同步优先级和频率
- 智能压缩 Agent 在备份前压缩不常用数据，节省存储空间

### Walrus 集成
- 完整备份以 `backup/{user_id}/full/v{N}.tar.enc` 加密存储
- 增量快照以 `backup/{user_id}/delta/{timestamp}.patch.enc` 存储
- 同步日志以 `backup/{user_id}/sync_log.json` 记录每次同步操作
- 设备注册信息以 Sui 链上对象存储，记录授权设备列表

### 前端 / 后端
- 前端：React + Electron 桌面客户端，常驻系统托盘
- 后端：Rust 高性能同步服务，处理增量计算和加密
- 关键页面：备份状态页、版本历史页、设备管理页、冲突解决页

## 技术架构

Agent 数据本地变更 → 变更检测 Agent 捕获事件 → 增量计算 Agent 生成补丁 → Seal 端到端加密 → 加密补丁写入 Walrus → Sui 链上更新版本号 → 其他设备检测到版本更新 → 从 Walrus 下载补丁 → 本地解密并应用 → 冲突检测 → 自动合并或提示用户。
